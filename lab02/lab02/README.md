## أسئلة للمناقشة

**1. لماذا نستخدم `concat` بدلاً من `push` عند تحديث مصفوفة الحالة؟**

لأن `push` تعدّل المصفوفة الأصلية مباشرة، وهذا يخالف مبدأ React في عدم تعديل الحالة مباشرة. أما `concat` فتُنشئ مصفوفة جديدة، مما يجعل React تكتشف التغيير وتعيد الرسم.

**2. لماذا نحتاج `[...votes]` عند التصويت بدلاً من تعديل المصفوفة مباشرة؟**

لأن `votes` مصفوفة في الحالة ولا يجب تعديلها مباشرة. `[...votes]` تنشئ نسخة جديدة منها، فيتعرّف React على التغيير ويعيد رسم المكوّن.

**3. ما الفرق بين `onClick={handleClick}` و `onClick={() => handleClick()}`؟**

- `onClick={handleClick}` تمرر الدالة مباشرة وتُستدعى عند الضغط.
- `onClick={() => handleClick()}` تنشئ دالة جديدة في كل رسم، وتُستدعى عند الضغط. تُستخدم حين نحتاج تمرير معاملات مثل `onClick={() => handleClick(id)}`.

**4. متى نستخدم حالة واحدة معقدة (كائن) ومتى نستخدم عدة حالات منفصلة؟**

نستخدم حالات منفصلة حين تكون القيم مستقلة عن بعض مثل `good`, `neutral`, `bad`. ونستخدم كائناً واحداً حين تكون القيم مرتبطة ببعض وتُحدَّث معاً مثل بيانات نموذج.
